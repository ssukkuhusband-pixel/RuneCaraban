# 룬 캐러밴 (프로토타입) - 기획서

## 한 줄 피치
모바일 가로 화면에서 룬 슬롯을 회전해 영웅 행동을 발동하고, 전투마다 영입/강화 보상을 선택해 파티를 키우는 수집형 로그라이트 전투 프로토타입.

## 타깃 / 플레이 세션
- 타깃: 짧은 전술 런과 수집 성장 루프를 선호하는 모바일 유저
- 세션: 4~8분(10개 스테이지 기준)

## 화면 방향 / 해상도
- 방향: 가로(landscape)
- 기준 해상도: `844 x 390` (CSS px), 비율 유지 스케일

## 핵심 재미
- 회전 결과가 전투 전술의 변수를 만든다.
- 영웅 영입이 파티 정체성과 전투 옵션을 확장한다.
- 3지선다 보상이 매 런의 빌드를 바꾼다.

## 코어 루프
1. 전투 노드 진입
2. 룬 슬롯 회전(3룬) 후 자동 해석
3. 적 턴 진행
4. 전투 승리
5. 보상 3개 중 1개 선택(강화/영입/유지력)
6. 다음 노드 이동(전투/정예/휴식 변주), 보스 클리어 또는 전멸까지 반복

## 전투 규칙 (프로토)
- 턴 흐름: 플레이어 회전 → 룬 3개 자동 해석 → 적 턴
- 승리 조건: 모든 적 처치
- 패배 조건: 출전 영웅 전원 전투 불능
- 영웅 궁극기 규칙:
  - 영웅 행동 시 에너지 획득
  - 에너지 100%일 때 해당 영웅 룬은 궁극기로 실행
  - 궁극기 발동 후 에너지 0%로 초기화
- 적 행동 규칙:
  - 각 적은 다음 행동 의도(단일/전체/방어/흡수)를 사전에 노출
  - 적 행동 후 다음 의도를 다시 굴림
  - 보스는 체력 50% 이하에서 격노(공격 상승 + 보호막 획득)
- 노드 규칙:
  - 총 10개 스테이지 진행, 노드 순서: 전투 → 전투 → 정예 → 전투 → 휴식 → 전투 → 정예 → 전투 → 휴식 → 보스
  - 정예 노드는 일반 노드 대비 적 체력/공격이 높고 추가 보상을 제공
  - 휴식 노드는 전투 없이 캠프 선택을 제공하고 다음 노드로 진행
- 파티 규칙:
  - 출전 파티 최대 4명
  - 수집한 영웅은 빈 슬롯이 있으면 즉시 출전 편성 가능
  - 출전 인원 가득 찬 상태에서 영입 선택 시 스탯 보상으로 전환

## 슬롯 규칙
- 슬롯은 3룬이며 왼쪽부터 순차 해석
- 룬 결과와 행동 주체 매핑:
  - 영웅 룬(`H1~H6`): 해당 영웅이 행동
  - 전술 룬(`T_ASSIST`, `T_BRACE`): 중립 보조 행동 발동
- 확장 표식:
  - `x2`: 해당 심볼 2회 행동
  - `↺`: 좌측부터 재해석(턴당 1회)
  - `▲`: 해당 심볼 위력 강화
  - `∞`: 해당 심볼 60% 위력 추가 발동
- 이중 페널티 방지:
  - 전투 불능 영웅 룬은 룬 풀에서 제외
  - 제외된 가중치는 전술 룬으로 재분배(무효 스핀 방지)
  - 생존 영웅이 1~2명일 때 영웅 룬 최소 개수 보정 적용
- 룬 시너지:
  - 쌍룬/삼중/전술 공명/연계 진형에 따라 해당 턴 보너스 적용
  - 보너스 상태는 HUD(턴 보너스 pill)로 즉시 노출

## 수집 레이어 (프로토)
- 영웅 풀 총 6명(공격수/결투가/마도사/수호자/치유사/궁수)
- 시작 시 2명 보유
- 보상의 영입 선택으로 컬렉션 및 출전 파티 확장(최대 4명)
- 최소 메타 성장 구현(유산 결정 기반 영구 단련 3종)

## 로그라이트 보상 규칙
- 전투 승리 후 보상 3개 랜덤 제시
  - 공격/치명/광역/보호막/전술 등 강화
  - 신규 영웅 영입
  - 유물 획득(런 상시 효과)
  - 회복/부활 유지력 선택지
- 1개만 선택 가능
- 선택 효과는 런 종료까지 유지
- 추가 시스템:
  - 턴 내 룬 해석 연계 배율(콤보) 적용
  - 전투 불능 아군 수 기반 공격 보정(이중 페널티 완화)
  - 룬 시너지 + 궁극기 에너지 루프 결합

## 성공 기준
1. 플레이어가 룬 결과와 실제 행동의 연결을 즉시 설명할 수 있다.
2. 영입 + 강화 선택으로 파티 성장 체감을 준다.
3. 모바일 가로 해상도에서 UI 겹침 없이 읽힌다.
4. 시작 → 보스 → 클리어/게임오버 루프가 끊기지 않는다.
5. 적 의도/룬 시너지 정보를 UI만 보고 파악할 수 있다.
6. 궁극기 충전/발동 타이밍을 UI만 보고 판단할 수 있다.
7. 노드 타입(전투/정예/휴식/보스)의 차이를 진행 중 이해할 수 있다.
8. 유물 획득에 따른 빌드 변화가 즉시 체감된다.

## 스코프 컷
- 계정 단위 서버 저장/동기화
- 상점/가챠/경제 시스템
- 복잡한 상태이상/원소 상성
- 외부 에셋 파이프라인

## 가정
- 터치 우선, 마우스 폴백 제공
- 텍스트 최소화 + 아이콘 중심 전달
- CSS 기반 경량 애니메이션으로 시각 완성도 확보
